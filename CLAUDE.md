## Architecture

**Next.js 16 App Router** with React 19, TypeScript, Tailwind CSS v4. 100% vegetarian restaurant website for Burger Minister (Sector 58, Noida) with online ordering.

### Routing

- `/` — Homepage: hero, testimonials, featured menu, FAQ, location, WhatsApp button
- `/menu` — Full menu with category filters (server page → `MenuClient.tsx` client component)
- `/cart` — Shopping cart with coupon codes
- `/checkout` — Two-step: details form → UPI payment modal → confirm order
- `/thankyou` — Post-order confirmation
- `/contact`, `/about`, `/catering` — Static-ish pages with forms
- `/api/order` — POST endpoint: sends order to Google Sheets webhook + Web3Forms email

### State Management

`src/context/CartContext.tsx` — React Context wrapping the app. Cart items + coupon state persist to `localStorage`. Two active coupon codes: `MINISTER05` (5%), `MINISTER38` (10% — hidden, shared verbally).

### Styling

Tailwind CSS v4 via `@tailwindcss/postcss` — no `tailwind.config.ts`. All custom theme (Material Design 3 dark palette) lives in `src/app/globals.css` as CSS variables. Key custom classes: `.btn-honeyed`, `.glass-nav`, `.ambient-shadow`, `.ghost-border`.

Primary color: `#e6c443` (gold). Fonts: Epilogue (headings) + Plus Jakarta Sans (body), loaded via Next.js Google Fonts in layout.

### Shared Data

- `src/lib/images.ts` — All Unsplash image URLs (menu items, banners, categories)
- `src/lib/faqs.ts` — FAQ data array (imported by both client `FAQ.tsx` and server `Schema.tsx`)

### SEO & Structured Data

- `src/components/Schema.tsx` — `LocalBusinessSchema` (FoodEstablishment) + `FAQSchema` JSON-LD, rendered in layout/homepage
- `src/app/sitemap.ts` + `src/app/robots.ts` — Auto-generated by Next.js
- Canonical domain: `https://burger-minister.com`
- Each route page sets its own `metadata` with canonical via `alternates`

## Key Patterns

- **Server/Client boundary**: Pages under `src/app/` are server components by default. Interactive parts are extracted into `"use client"` components (e.g., `MenuClient.tsx`, `FeaturedMenu.tsx`, `ContactForm.tsx`). Never import a `"use client"` file's exports into a server component for data — use `src/lib/` for shared data instead.
- **Menu variants**: Items with Half/Full or M/L sizes use a `variants` array instead of separate entries. Cart stores them as `"Item Name — Half"` format.
- **No database**: Orders go to Google Sheets (Apps Script webhook) + email (Web3Forms). No auth, no user accounts.
- **Images**: All from Unsplash CDN via `next/image` with remote pattern configured in `next.config.ts`.

## Environment Variables

```
GOOGLE_SHEET_WEBHOOK=   # Google Apps Script URL for order submissions
WEB3FORMS_KEY=          # Web3Forms API key for email notifications
```

Must be set on Vercel separately (not just `.env.local`).
